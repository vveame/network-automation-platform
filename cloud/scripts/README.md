# Cloud Scripts

## 1. Purpose

This folder contains helper scripts used by Jenkins and by the DevOps server to synchronize runtime artifacts between the local platform and AWS S3.

The scripts support the cloud-backed architecture of the platform:

```text
Jenkins workspace
      ↓
AWS S3 artifact bucket
      ↓
/var/lib/pfe-dashboard local cache
      ↓
Flask dashboard
```

AWS S3 is the durable source of truth for generated artifacts.

The local dashboard folder is only a synchronized cache.

---

## 2. Main Scripts

### `upload-validation-artifacts-s3.sh`

Uploads Ansible validation outputs to S3.

Source folder:

```text
ansible/outputs/
```

S3 destinations:

```text
validation-artifacts/<build-label>/
latest/validation-artifacts/
```

These files are later displayed by the Flask dashboard under the validation page.

---

### `upload-prometheus-snapshot-s3.sh`

Uploads the Prometheus snapshot exported by Jenkins.

Source folder:

```text
monitoring/outputs/latest/
```

S3 destinations:

```text
metrics-snapshots/<build-label>/
latest/metrics/
```

These metrics are used by the rule-based analyzer, ML feature builder, and dashboard monitoring page.

---

### `sync-dashboard-cache-from-s3.sh`

Synchronizes the latest S3 outputs into the local dashboard cache.

Source S3 folders:

```text
latest/validation-artifacts/
latest/metrics/
latest/analyzer/
latest/ml/
latest/ml-dataset/
latest/remediation/
```

Local destinations:

```text
/var/lib/pfe-dashboard/outputs/
/var/lib/pfe-dashboard/metrics/latest/
/var/lib/pfe-dashboard/analyzer/latest/
/var/lib/pfe-dashboard/ml/latest/
/var/lib/pfe-dashboard/ml/data/
/var/lib/pfe-dashboard/remediation/latest/
```

This script allows the Flask dashboard to show the latest Jenkins-generated state.

---

## 3. S3 Artifact Structure

The platform stores two types of folders in S3:

```text
<artifact-type>/<build-label>/ = historical archive
latest/<artifact-type>/        = latest dashboard-ready version
```

Historical folders:

```text
validation-artifacts/<build-label>/
metrics-snapshots/<build-label>/
processed-summaries/<build-label>/
anomaly-results/<build-label>/
ml-datasets/<build-label>/
ml-results/<build-label>/
remediation-results/<build-label>/
```

Latest folders:

```text
latest/validation-artifacts/
latest/metrics/
latest/analyzer/
latest/ml/
latest/ml-dataset/
latest/remediation/
```

## 4. Validation Artifacts

Validation artifacts are generated by Ansible during the Jenkins pipeline.

They include:

```text
validation-summary.txt
index.html
jenkins-inventory.json
FRR validation reports
OVS validation reports
DMZ service reports
Security validation reports
End-to-end validation reports
```

They are uploaded to:

```text
validation-artifacts/<build-label>/
latest/validation-artifacts/
```

---

## 5. Metrics Snapshots

Prometheus snapshots are exported during each Jenkins pipeline run.

They include metric JSON files such as:

```text
up.json
blackbox_probe_success.json
blackbox_probe_duration_seconds.json
node_cpu_usage_percent.json
node_memory_used_percent.json
snmp_up.json
snmp_if_oper_status.json
snmp_if_in_errors_rate_5m.json
snmp_if_in_discards_rate_5m.json
```

They are uploaded to:

```text
metrics-snapshots/<build-label>/
latest/metrics/
```

---

## 6. Rule-Based Analyzer Outputs

The rule-based analyzer consumes validation reports and monitoring snapshots.

It generates:

```text
summary.json
decision.json
analysis-report.txt
```

These files are uploaded to:

```text
processed-summaries/<build-label>/
anomaly-results/<build-label>/
latest/analyzer/
```

---

## 7. ML Analyzer Outputs

The ML analyzer consumes historical Prometheus metric windows.

It generates a feature dataset:

```text
latest_features.csv
```

It also generates ML decision files:

```text
ml-decision.json
ml-scores.csv
```

The Isolation Forest model is persisted as:

```text
isolation_forest.joblib
feature_columns.json
training_metadata.json
```

S3 destinations:

```text
ml-datasets/<build-label>/
ml-results/<build-label>/
ml-models/latest/
latest/ml-dataset/
latest/ml/
```

The ML result is advisory and does not directly trigger infrastructure-changing remediation.

---

## 8. Final Hybrid Decision Outputs

After the rule-based analyzer and ML analyzer finish, Jenkins merges both decisions.

Generated files:

```text
final-decision.json
final-decision-report.txt
```

S3 destinations:

```text
anomaly-results/<build-label>/
processed-summaries/<build-label>/
latest/analyzer/
```

The final decision determines whether remediation is allowed.

---

## 9. Safe Remediation Outputs

The safe remediation runner consumes:

```text
final-decision.json
```

It generates:

```text
remediation-plan.json
remediation-report.txt
```

Depending on the mode, files are stored under:

```text
plan/
apply/
```

S3 destinations:

```text
remediation-results/<build-label>/
anomaly-results/<build-label>/remediation/
latest/remediation/
```

Plan mode does not execute commands.

Apply mode requires explicit confirmation:

```text
REMEDIATION_MODE=apply
CONFIRM_APPLY=true
```

Infrastructure-changing actions also require:

```text
remediation_allowed=true
```

## 11. Manual Sync Example

To manually synchronize the dashboard cache:

```bash
ARTIFACTS_BUCKET="<bucket-name>" \
AWS_REGION="eu-north-1" \
./cloud/scripts/sync-dashboard-cache-from-s3.sh
```

After synchronization, check:

```bash
find /var/lib/pfe-dashboard -maxdepth 5 -type f | sort
```

## 12. Manual Validation of S3 Content

List latest analyzer outputs:

```bash
aws s3 ls s3://<bucket-name>/latest/analyzer/ --region eu-north-1
```

List latest ML outputs:

```bash
aws s3 ls s3://<bucket-name>/latest/ml/ --region eu-north-1
```

List latest remediation outputs:

```bash
aws s3 ls s3://<bucket-name>/latest/remediation/ --recursive --region eu-north-1
```

List historical build artifacts:

```bash
aws s3 ls s3://<bucket-name>/anomaly-results/<build-label>/ --recursive --region eu-north-1
```

---

## 13. Safety Notes

These scripts only upload and synchronize artifacts.

They do not directly modify the network.

Network-changing actions are handled only by the safe remediation runner and only through Jenkins-controlled stages.

The safety model is:

```text
ML-only signal = advisory, human review
Rule-confirmed anomaly = remediation candidate
Apply mode = explicit confirmation required
Infrastructure-changing action = allowed only if remediation_allowed=true
```

---

## 14. Operational Summary

The cloud scripts support the full automation chain:

```text
Ansible reports
      ↓
Prometheus snapshots
      ↓
Rule analyzer outputs
      ↓
ML analyzer outputs
      ↓
Final hybrid decision
      ↓
Safe remediation outputs
      ↓
AWS S3
      ↓
Flask dashboard cache
```

They are a key part of the platform because they separate temporary Jenkins execution from durable cloud storage and dashboard visualization.
